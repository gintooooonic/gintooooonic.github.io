어노테이션은 주석처럼 프로그램에 영향을 미치지 않으면서
컴파일러와 같은 다른 프로그램에 유용한 정보를 제공하는 기능을 한다.

예를 들어 `@Test` 어노테이션을 메서드 앞에 붙여
테스트 프로그램에게 '해당 메서드를 테스트해야 한다'라는 것을
알릴 수 있다.

```java
@Test
public void method() { ... }
```

java.lang.annotation 패키지에 포함된
JDK 표준 어노테이션과 다른 프로그램에서 제공하는 어노테이션이 있다.

## 표준 어노테이션

- `@Override` : 컴파일러에게 오버라이딩하는 메서드임을 알린다.
  컴파일러는 같은 이름의 메서드가 조상에 있는지 확인하고, 없으면 에러를 발생시킨다.
- `@Deprecated` : 더 이상 사용하지 않을 것을 권장하는 대상에 붙인다.
  이것이 붙은 메서드 등을 사용하면 컴파일러가 경고 메시지를 출력한다.
- `@SuppressWarnings` : 컴파일러의 특정 경고 메시지가 나타나지 않게 해준다.
- `@SafeVarargs` : 제네릭스 타입의 가변 인자에 사용한다.
- `@FunctionalInterface` : 함수형 인터페이스임을 알린다.
  컴파일러가 함수형 인터페이스를 올바르게 선언했는지 확인한다.
- `@Native` : native 메서드에서 참조되는 상수 앞에 붙인다.
- `@Target` : 어노테이션이 적용 가능한 대상을 지정하는데 사용한다.
- `@Documented` : 어노테이션 정보가 javadoc으로 작성된 문서에 포함되게 한다.
- `@Inherited` : 어노테이션이 자손 클래스에 상속되도록 한다.
- `@Retention` : 어노테이션이 유지되는 범위를 지정한다.
- `@Repeatable` : 어노테이션을 반복해서 적용할 수 있게 한다.

`@Target`부터 `@Repeatable`까지는 메타 어노테이션으로서,
어노테이션의 적용 대상이나 유지 기간 등을 지정하는
'어노테이션의 어노테이션'이다.

## 사용자 정의 어노테이션

```java
@interface 이름 {
  타입 요소이름();
}
```

- 요소의 타입은 기본형, String, enum, 어노테이션, Class만 허용
- 요소의 매개 변수를 선언할 수 없음
- 예외를 선언할 수 없음
- 요소의 타입을 타입 매개변수(제네릭)로 정의할 수 없음
