클래스 사이의 관계를 설정하는 방법으로
'상속'과 '포함'을 제시한다.

## 상속

상속(Inheritance)이란 기존의 클래스를 재사용하여
새로운 클래스를 작성하는 것.
클래스 사이에 관계를 맺어 주고
코드를 재사용할 수 있다.

상속해주는 클래스를 조상 클래스,
상속 받는 클래스를 자손 클래스라 일컫는다.

```java
class Child extends Parent { ... }
```

Child 클래스는 Parent 클래스를 상속한 자손 클래스이다.
Parent 클래스는 Child 클래스의 조상 클래스이다.

자손 클래스는 조상 클래스의
모든 멤버를 상속받는다.
조상 클래스가 갖는 모든 멤버 변수와 메서드를
자손 클래스도 갖게 된다.
(멤버만을 상속받는다. 생성자와 초기화 블럭은
해당되지 않는다.)

> 자손 클래스가 조상 클래스의 모든 멤버를
> 상속받는다는 내용에 의구심을 가질 수 있을 것 같다.
> 해당 멤버의 접근 제어자가 `private`나 `default`라면?
> 그래도 해당 멤버는 상속된다.
> 단지 자손 클래스에서 접근이 제한될 뿐이다.

> 자바는 다중 상속을 지원하지 않는다.

> 모든 자바 클래스의 최상위 조상은 Object 클래스이다.

## 포함

클래스 사이의 관계를 형성하며
코드를 재사용하는 또 하나의 방법으로
포함(Composite) 관계를 꼽을 수 있다.

포함 관계란 말 그대로 하나의 클래스가
다른 클래스 타입의 참조변수를 멤버로 갖는 것을 말한다.

```java
class Circle {
  // Point 타입의 참조변수 p를 멤버로 포함
  Point p = new Point();
  ...
}
```

## 상속이냐, 포함이냐

클래스 사이에 상속 관계를 만드는 것이 적절한지,
포함 관계를 만드는 것이 적절한지 판단할 수 있는
팁.

### 1. 상속, IS-A

IS-A. 말 그대로 `~은 ~이다`라는 문장이 성립한다면
상속 관계를 맺어주는 것이 적절할 수 있다.

```
SportsCar는 Car이다. (O)
SportsCar는 Car를 가지고 있다. (X)
```

### 2. 포함, HAS-A

HAS-A. `~은 ~을 가지고 있다`라는 문장이 성립한다면
포함 관계를 맺을 수 있다.

```
Circle은 Point를 가지고 있다. (O)
Circle은 Point이다. (X)
```

## 상속에 대한 이야기

### 오버라이딩

조상 클래스로부터 상속받은 메서드의 내용을 변경하는 것.
오버라이딩을 위해서는 접근 제어자와 예외를 제외한
메서드의 선언부가 일치해야 한다.

- 이름이 같아야 한다.
- 매개변수가 같아야 한다.
- 반환타입이 같아야 한다.
- 접근 제어자는 조상 클래스의 메서드보다 좁은 범위로 변경할 수 없다.
- 조상 클래스의 메서드보다 많은 수의 예외를 선언할 수 없다.
- 인스턴스 메서드를 클래스 메서드로 혹은 그 반대로 변경할 수 없다.

```java
class Point {
  int x, y;
  String getLocation() {
    return x + ", " + y;
  }
}

class Point3D extends Point {
  int z;
  String getLocation() {
    return x + ", " + y + ", " + z;
  }
}
```

### super

`super`은 조상 클래스로부터 상속받는 멤버를 참조하는데 사용된다.
조상 클래스와 자손 클래스에 같은 이름을 갖는 멤버가 있고,
이 둘을 구별해서 접근해야 하는 경우 각각 `super`와 `this`를 사용할 수 있다.

```java
class Point {
  int x, y;
  String getLocation() {
    return x + ", " + y;
  }
}

class Point3D extends Point {
  int z;
  String getLocation() {
    return super.getLocation() + ", " + z;
  }
}
```

### super()

`super()`은 조상 클래스의 생성자이다.
Object 클래스를 제외한 모든 클래스는
첫 줄에 `this()` 또는 `super()`을 호출해야 한다.
그렇지 않으면 컴파일러가 자동으로 `super();`을 추가한다.
