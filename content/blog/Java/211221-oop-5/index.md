조상 클래스 타입의 참조변수로
자손 클래스의 인스턴스를 참조할 수 있다.
(반대는 에러가 발생한다.)

```java
Car c = new SportsCar();
```

실제 객체는 `SportsCar`의 인스턴스이지만,
`SportsCar`의 모든 멤버를 사용할 수 없다.
타입이 `Car`이기 때문에, 접근할 수 있는 건
`Car` 클래스에 정의된 멤버들뿐이다.

## 업캐스팅과 다운캐스팅

- 업캐스팅 : 자손 타입의 참조변수를 조상 타입의 참조변수로
- 다운캐스팅 : 조상 타입의 참조변수를 자손 타입의 참조변수로

업캐스팅의 경우 형변환을 생략할 수 있다.
하지만 다운캐스팅은 캐스팅 연산자를 사용해 명시적으로
캐스팅하여야 한다.

이러한 참조변수의 캐스팅은 오로지 참조변수의 타입을 변경하는 것이므로
인스턴스 자체에는 영향을 미치지 않는다.

## 참조변수와 인스턴스

조상 클래스와 자손 클래스에
같은 이름의 멤버가 정의된 경우,
조상 타입의 참조변수로 자손 타입의 인스턴스를 참조하는 것은
메서드와 멤버변수의 관점에서 조금 다른 동작이 발생한다.

메서드의 경우 참조변수의 타입에 상관 없이 항상
실제 인스턴스의 메서드가 호출된다.

멤버변수의 경우 조상 클래스에 선언된 멤버변수가 사용된다.
