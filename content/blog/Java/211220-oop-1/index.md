## 객체지향이란

> 실제 세계는 사물(객체)로 이루어져 있으며, 발생하는 모든 사건들은 사물간의 상호작용이다.

실제 사물의 속성과 기능을 분석한 다음, 데이터(변수)와 함수로 정의한다.

## 객체, 클래스, 인스턴스

- 객체 (Object) : 실제의 사물 또는 개념. 속성과 기능을 포함한다.
- 클래스 (Class) : 객체의 정의, 객체를 생성하는데 사용하는 틀(설계도)
- 인스턴스 (Instance) : 클래스로부터 만들어진 객체

> 클래스로부터 만들어진 객체를 그 클래스의 인스턴스라고 한다. 객체지향을 처음 공부할때 객체와 인스턴스를 혼동했던 기억이..
>
> - 책상은 인스턴스이다. (X)
> - 책상은 객체이다. (O)
> - 책상은 책상 클래스의 인스턴스이다. (O)

## 객체의 생성

아래는 `TV` 클래스의 예제이다.

```java
class TV {
  String color;
  boolean power;
  int channel;

  void power() { power = !power; }
  void channelUp() { ++channel; }
  void channelDown() { --channel; }
}
```

언급했듯 클래스는 객체를 생성하는 설계도의 역할을 한다.
클래스를 사용해 객체를 생성한다.

```java
TV t = new TV();
t.power();
t.channel = 5;
t.channelUp();
t.power();
```

`t`는 `TV` 클래스의 한 인스턴스를 가리키는 참조변수이다.
즉, 해당 인스턴스의 주소값을 저장한다.
`t`를 통해 인스턴스가 갖고 있는 속성과 기능을 사용할 수 있다.

여러 개의 객체가 필요하다면 객체 배열을 생성할 수도 있다.

```java
// TV 배열을 생성
TV[] tvArr = new TV[3];

// 각 TV 객체를 생성
for (int i = 0; i < tvArr.length; ++i)
  tvArr[i] = new TV();
```

객체 배열을 생성하는 것에 그치는게 아니라,
배열의 원소에 해당하는 각 객체들을 직접 생성해주어야 한다는 것을 기억하자.
